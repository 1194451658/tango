private helper when trying to invoke functions on the proxy, this method will be called wraps the variable arguments into a table and transmits them to the server

Parameters

  • self: the proxy
  • ...: variable argument list
__index (self, key)
private helper called when dot operator is invoked on proxy to access function or table

Parameters

  • self: the parent proxy
  • key: the proxy / remote table key to index
_proxy (socket, functionpath)
private helper. create a rpc proxy which operates on the socket provided (socket is not allowed to be copas.wrap'ed) functionpath is used internally and should not be assigned by users (addresses the remote function and may look like "a.b.c")

Parameters

  • socket:
  • functionpath:
client (adr, port, options)
returns a proxy to the specified client invoke remote functions on the returned variable

Parameters

  • adr: the server address, may be server name e.g. www.horst.de
  • port: the port on which the server listens (default 12345)
  • options:

Usage

  • c = tango.connect('localhost'); c.greet('horst') it is also possible to functions inside tables, like
  • c.utils.greetall()

Return value:

a proxy to the server global table
copasserver (socket)

serialize (tbl)
default serializer implementation copied from http://lua/users.org/wiki/TableUtils may be overwritten for custom serialization (function must take a table and return a string)

Parameters

  • tbl: the table to be serialized

Usage:

tango.serialize = table.marshal (using lua-marshal as serializer)

Return value:

the serialized table as string
serve (port)
